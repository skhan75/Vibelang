# Docs Usability Walkthrough Checklist (Canonical)

Date: 2026-02-22

Run this each RC cycle and capture pass/fail in:
`reports/docs/documentation_quality.md`.

## New Team Onboarding Path

- [ ] Can a new team run chapter 1 setup without undocumented prerequisites?
- [ ] Can a new team build and run the first program from docs only?
- [ ] Are error messages in walkthrough steps reproducible and understandable?

## Daily Developer Workflow

- [ ] `check/build/run/test/fmt/lint/index/lsp` docs are internally consistent.
- [ ] Debug/profiling docs link to current command contracts.
- [ ] Migration and compatibility guidance is discoverable from chapter index.

## Operations And Release

- [ ] Security/support/release policy docs are linked from docs entry points.
- [ ] Known limitations and breaking changes paths are discoverable.
- [ ] RC/GA evidence documents referenced from release docs exist and are current.
