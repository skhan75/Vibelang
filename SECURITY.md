# Security Policy

## Reporting a vulnerability

If you believe you have found a security vulnerability in VibeLang, please **do not** open a
public GitHub issue.

- **Report privately**: email `security@thevibelang.org` with details and a minimal reproduction.
- **Include**: affected versions, impact, steps to reproduce, and any suggested mitigations.

We will acknowledge receipt and begin triage as soon as possible.

## Disclosure and response

VibeLang follows coordinated disclosure and a structured CVE response workflow:

- Disclosure policy: `docs/security/disclosure_policy.md`
- CVE response workflow: `docs/security/cve_response_workflow.md`

## Supported versions

Security fixes are released for supported versions according to the project’s release policies.
