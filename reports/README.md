# Reports

This directory contains generated reports from CI, benchmarks, and release
processes. Most files here are machine-generated and updated by automation.

## Directory structure

| Directory | Contents | Audience |
|---|---|---|
| `benchmarks/third_party/` | PLB-CI benchmark results, analysis, and deltas | Maintainers, performance reviewers |
| `v1/` | Release readiness dashboards and checklists | Release managers |
| `docs/` | Documentation quality reports | Doc maintainers |
| `examples/` | Example parity and coverage reports | Contributors |
| `phase*/` | Per-phase development reports | Maintainers |

## Benchmark reports

Raw benchmark data (`results.json`, task logs, history snapshots) is excluded
from version control via `.gitignore` because it contains machine-specific
metadata and is not validated for public publication. See
`benchmarks/third_party/APPLE_TO_APPLE_BENCHMARK_POLICY.md` for the
publication policy.

The human-readable summary (`benchmarks/third_party/full/summary.md`) is
published only after all publication blockers are resolved.

## Regenerating reports

Benchmark reports are regenerated by CI or locally via:

```bash
python3 tooling/metrics/collect_third_party_benchmarks.py --profile full
```

See `benchmarks/third_party/CLOUD_REPRODUCIBILITY.md` for the full
reproducibility recipe.
